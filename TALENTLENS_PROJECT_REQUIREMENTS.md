## 1. Project Overview

**TalentLens** is an AI-powered recruitment intelligence platform targeting HR teams, particularly in the UAE market. It streamlines the end-to-end hiring workflow from job profile creation through candidate CV ingestion, AI scoring, shortlisting, and WhatsApp-based candidate engagement.

### Core Value Proposition
- Eliminate manual CV screening with automated AI scoring
- Enforce configurable mandatory/preferred qualification gates
- Enable side-by-side comparison of shortlisted candidates
- Integrate WhatsApp as the primary candidate communication channel
- Provide an intelligent chat assistant to answer questions about candidates

### Target Market Signals (from mock data)
- UAE-based construction, engineering, and project management roles
- Multi-national candidate pools (Emirati, Indian, British, Egyptian, Japanese, Nigerian, Malaysian, Saudi, Moroccan)
- UAE-specific compliance: Right to Work, Driving License, Arabic language
- Cities: Dubai, Abu Dhabi

## 3. Data Models & Type System

All types are defined in `/home/user/CV-viewer/types/index.ts`.

### 3.1 Job Profile

```typescript
JobProfile {
  id, title, department, location, openings, status: JobStatus,
  description, createdAt, updatedAt,
  mandatoryCriteria: MandatoryCriteria,
  preferredCriteria: PreferredCriteria,
  scoringWeights: ScoringWeights,
  stats: JobStats
}

MandatoryCriteria {
  degreeRequired, degreeLevel, degreeFields[], minYearsExperience,
  maxYearsExperience, certifications[], drivingLicenseRequired,
  rightToWorkUAE, allowedNationalities[]
}

PreferredCriteria {
  skills[]: { id, name, weight(1-5) },
  industryBackground[], employerType[], languages[], projects[]
}

ScoringWeights {
  yearsExperience, technicalSkills, industryBackground,
  certifications, languages    // must sum to 100
}

JobStats {
  uploaded, scored, shortlisted, eliminated, avgScore,
  scoreDistribution: { range, count }[]
}
```

**Status values**: `active` | `paused` | `closed` | `draft`

### 3.2 Candidate

```typescript
Candidate {
  id, jobProfileId, name, email, phone, nationality, age, languages[],
  currentTitle, currentCompany, yearsExperience,
  degreeLevel, degreeField,
  education[]: EducationEntry,
  experience[]: ExperienceEntry,
  skills[]: CandidateSkill,
  certifications[], status: CandidateStatus,
  finalScore(0-100),
  criterionScores: Record<string, CriterionScore>,
  aiSummary, eliminationReason,
  cvFilePath, extractionConfidence,
  shortlistedAt, rejectedAt, createdAt
}

CriterionScore { score, justification, passFail? }
CandidateSkill  { name, matched: boolean, confidence(0-1) }
```

**Status pipeline**: `uploaded` → `extracting` → `scoring` → `scored` → `shortlisted` | `under_review` | `rejected` | `eliminated`

### 3.3 WhatsApp

```typescript
WhatsAppThread {
  id, candidateId, candidateName, candidateInitials, jobProfileId,
  jobTitle, phoneNumber, status: ThreadStatus,
  lastMessage, lastMessageAt, unread, candidateScore,
  messages: WhatsAppMessage[]
}

WhatsAppMessage {
  id, threadId, direction: 'inbound' | 'outbound',
  content, messageType: 'template' | 'free_text' | 'screening' | 'ai_draft',
  sentAt, deliveredAt, readAt, isSystemEvent
}
```

**Thread status**: `screening` | `follow_up` | `closed`

### 3.4 Supporting Types

```typescript
Notification { id, type, title, message, read, createdAt, href }
FilterState   { scoreMin, scoreMax, statuses[], scoreTier, missingCriteria[], sortBy }
CompareSelection { jobProfileId, candidateIds[] }  // max 4

UploadedFile  { id, name, size, type, status: UploadStatus, progress, error }
```

## 4. Page & Route Inventory

### 4.1 Dashboard — `/`
- **Purpose**: High-level KPI overview + job profile grid
- **Metrics**: Active Jobs · Total Candidates · Shortlisted · Avg Time to Shortlist
- **Data**: `apiClient.jobs.list()`
- **Features**: Animated card grid, error retry, loading skeletons
- **Nav**: → `/jobs/new`, → each job's upload/results pages

### 4.2 Job List — `/jobs`
- **Purpose**: Dedicated full-page job profile list
- **Data**: `apiClient.jobs.list()`
- **Same components as dashboard, no stats row**

### 4.3 New Job Profile — `/jobs/new`
- **Purpose**: 5-step guided job profile creation wizard
- **Steps**:
  1. Role Basics (title, department, location, openings, description)
  2. Mandatory Criteria (degree, experience range, certs, UAE compliance)
  3. Preferred Criteria (skills with weights, industry, employer, languages, projects)
  4. Scoring Weights (allocate 100 points across 5 criteria with ring visualizer)
  5. Review & Save (collapsible read-only summary, activate or draft)
- **Data**: `apiClient.jobs.create()`
- **Features**: Step validation guards, animated slide transitions, desktop sidebar + mobile tabs, template suggestions

### 4.4 Edit Job Profile — `/jobs/[id]/edit`
- **Status**: PLACEHOLDER — incomplete
- **Note**: "Pre-population connected in Phase 2"
- **Needs**: Same multi-step form pre-populated with existing job data

### 4.5 CV Upload — `/jobs/[id]/upload`
- **Purpose**: Bulk CV upload and simulated processing
- **Accepted formats**: PDF, DOCX (max 50 files)
- **Status pipeline**: queued → extracting (30%) → scoring (70%) → done | failed
- **Features**: Drag-and-drop zone, progress bars, real-time status, file removal (queued only), 12.5% simulated failure rate
- **Note**: Processing is currently **simulated with timeouts** — no real backend call
- **Nav**: → `/jobs/[id]/results` when complete

### 4.6 Candidate Results — `/jobs/[id]/results`
- **Purpose**: Filtered, sorted candidate list for a job
- **Data**: `apiClient.candidates.listByJob(id)`
- **Filter sidebar** (desktop) / drawer (mobile):
  - Score range sliders
  - Status checkboxes
  - Score tier radio buttons
  - Sort: score | date | experience
- **Features**: Compare checkbox (sticky bar appears at 2+ selected), eliminated candidates sorted last, local status updates
- **State**: `useFilterStore`, `useCompareStore`
- **Nav**: → `/jobs/[id]/upload`, → `/jobs/[id]/compare`, → `/jobs/[id]/candidates/[id]`

### 4.7 Candidate Comparison — `/jobs/[id]/compare`
- **Purpose**: Side-by-side table of 2–4 candidates
- **Data**: From `useCompareStore` + `allMockCandidates`
- **Table rows**: Overall score · Mandatory criteria · All scoring criteria · AI Summary
- **Features**: Sticky header + left column, color-coded best-value highlighting, Shortlist / WhatsApp / Reject actions per column
- **Note**: Candidate data currently sourced from mock, not API

### 4.8 Candidate Profile — `/jobs/[id]/candidates/[candidateId]`
- **Purpose**: Full candidate profile + scoring breakdown + chat
- **Data**: `apiClient.candidates.get(candidateId)`
- **Sections**: Header (photo, basic info) · Contact (reveal toggles) · Education timeline · Experience timeline · Skills (with match highlight) · Certifications · Scoring accordion (per-criterion with justification) · CV preview (placeholder)
- **Features**: Desktop split layout (profile + `ChatPanel`), expandable scoring justification, Shortlist / WhatsApp / Reject actions

### 4.9 WhatsApp Inbox — `/whatsapp`
- **Purpose**: Full WhatsApp-style messaging hub
- **Data**: `apiClient.whatsapp.threads()`
- **Features**: Tab filter (All / Screening / Replied / Closed), search by name/job, thread list with unread indicators, message thread with bubbles + system events, AI draft generation, message templates, optimistic updates, `ScoreGauge` in thread header

## 6. State Management

Three independent Zustand stores in `/home/user/CV-viewer/store/index.ts`:

### NotificationStore
```
notifications[]        — initialized from mockNotifications
markRead(id)           — mark single as read
markAllRead()          — mark all as read
addNotification(n)     — push to front of list
```

### CompareStore
```
selection: { jobProfileId, candidateIds[] } | null
toggleCandidate(jobId, candidateId)   — add/remove, max 4, reset on job switch
clearSelection()
```

### FilterStore
```
filters: { scoreMin, scoreMax, statuses[], scoreTier, missingCriteria[], sortBy }
setFilter(key, value)   — type-safe setter
resetFilters()          — restore defaults
```

**Note**: No store persistence (no localStorage/sessionStorage). All state resets on page refresh.

## 11. Identified Gaps & Missing Features

### 11.1 Critical Missing (Blockers for Production)

1. **Authentication & Authorization** — No login, session, or role-based access. The entire app is open.
2. **Real CV Processing Backend** — Upload page simulates processing. A backend pipeline (extraction → scoring) is required.
3. **Real WhatsApp API** — No WhatsApp Business API integration. All messages are optimistic UI only.
4. **CV File Storage** — CV file paths are stored as strings but no S3/storage backend is connected.
5. **Edit Job Profile** — Placeholder page with no implementation.

### 11.2 Functional Gaps

6. **Form Validation** — Scoring weights not validated to sum to 100 before save. No field-level errors on wizard steps.
7. **Candidate Comparison via API** — Compare page uses `allMockCandidates` instead of fetching live data.
8. **Chat AI Responses** — Hardcoded string responses, not dynamically generated from actual candidate data.
9. **Notification Polling** — Notifications are static; no real-time updates.
10. **Job Clone / Archive** — Menu items present but non-functional (`#` hrefs).
11. **Skill Drag & Drop** — Reorder UI exists but `@dnd-kit` integration is not wired.
12. **Template Loading** — Step 1 detects template matches but does not pre-fill fields.
13. **CV Preview** — react-pdf is installed but not connected in candidate profile.

### 11.3 UX / Polish Gaps

14. **Mobile Nav** — Navigation drawer does not close when a link is tapped.
15. **Store Persistence** — Filter state, compare selection reset on page refresh.
16. **Error Differentiation** — Generic `API Error {status}` for all failures; no user-friendly messages.
17. **Accessibility** — Limited ARIA labels, keyboard navigation, and screen reader support.
18. **Pagination / Virtualization** — No pagination or virtual scroll for large candidate lists.
19. **Loading States for Stores** — No global pending/loading state; async operations lack feedback.
20. **Dropdown Close on Select** — Nationality dropdown and some menus don't close after selection.

### 11.4 Backend API Surface Gaps

21. `POST /api/jobs/:id/upload` — CV file upload endpoint not defined in client
22. `GET /api/jobs/:id/upload-status` — Polling endpoint for processing status
23. `POST /api/candidates/:id/shortlist` / `reject` — Dedicated action endpoints
24. `DELETE /api/jobs/:id` — Archive/delete job
25. Request interceptors for auth token injection
26. Error envelope schema for structured backend errors

## 12. Technical Debt & Code Quality

### Positives
- Strict TypeScript throughout with well-defined domain types
- Clear separation: pages, components, lib, store, types
- Consistent dark theme via CSS custom properties
- Reusable UI primitives (EmptyState, ErrorState, Skeleton, StatusBadge)
- Framer Motion used appropriately for micro-interactions
- Zustand stores are focused and independent (good for re-render performance)

### Issues
- **Hardcoded lists**: Departments, certifications, nationalities, industries, languages are inline arrays — should be configuration or API-driven
- **Mock data in comparison page**: `allMockCandidates` imported directly in `/compare` page instead of using API client
- **Inline styles**: `Step2MandatoryQuals` uses inline CSS for dual-range slider — not theme-aware
- **No error boundaries**: No React error boundaries wrapping route segments
- **No request caching**: Every navigation triggers a fresh API call (no React Query, SWR, or Next.js cache)
- **`next.config.ts` is empty**: No image domains, redirects, or security headers configured
- **No environment variable validation**: `NEXT_PUBLIC_USE_MOCK` and `NEXT_PUBLIC_API_URL` not validated at startup
- **Unique ID generation**: `Date.now()` used for skill IDs — not collision-safe in rapid input
